7. 권장 개발 스택 (Flutter 기반)

7.1 클라이언트 (Flutter App) 1. Flutter: iOS/Android 단일 코드베이스 2. 상태관리: Riverpod(권장) 또는 BLoC 3. 네트워크: Dio + Retrofit(옵션), 실패 재시도/타임아웃 표준화 4. 로컬 저장: Drift(권장) 또는 Hive (오프라인/캐시, 연속 달성/루틴 임시 저장) 5. 푸시/알림: FCM + 로컬 알림 플러그인(스케줄 알림) 6. 분석/실험: Firebase Analytics(또는 Amplitude), Remote Config/A-B(옵션) 7. 결제:
• iOS: StoreKit
• Android: Google Play Billing
• 서버 검증(영수증/구독 상태) 필수 8. 에러/성능 모니터링: Sentry(권장) + Firebase Crashlytics(선택)

7.2 백엔드 (BFF/API) + 데이터

권장 구조: “모바일 앱 + BFF(API 서버) + 외부 서비스(AI, 결제검증)” 1. API 서버: NestJS(Node.js) 또는 FastAPI(Python)
• NestJS 장점: 타입/모듈화/가드/인터셉터로 확장 용이
• FastAPI 장점: AI/리포트 생성 파이프라인과 궁합 좋음 2. DB: PostgreSQL 3. ORM: Prisma(NestJS) 또는 SQLAlchemy(FastAPI) 4. 캐시/레이트리밋: Redis (AI 호출 비용/남용 방지에 중요) 5. 인증: JWT + Refresh Token(권장) / 소셜 로그인(Apple/Google) 단계적 도입 6. 스토리지: S3 호환(예: Cloudflare R2/AWS S3) — 프로필/이미지/리포트 첨부 대비 7. 검색/집계(옵션): Elastic/Opensearch는 Phase2 이후 고려(초기엔 SQL + materialized view로 버팀)

7.3 AI 코칭/리포트 (비용/품질/프라이버시 관점) 1. LLM Provider: OpenAI/Anthropic 중 택1 + 추후 멀티벤더(리스크 헤지) 2. 프롬프트/가드레일:
• 테마(Faith/Universal), 톤(F/T) 분리 템플릿
• 금칙어/민감정보 마스킹(신앙 데이터 포함 가능성) 3. 응답 성능:
• 스트리밍 응답(UX 개선)
• SLA 목표(예: p95 4~6초) 설정 + 타임아웃/재시도 정책 4. 비용 제어:
• 대화 요약/메모리(서버 저장) + 토큰 절감
• 일일 무료 쿼터/레이트리밋/유료 우선 처리 큐

7.4 인프라/운영 1. 배포: Docker + (ECS/Kubernetes/Cloud Run 중 택1) 2. CI/CD: GitHub Actions 3. 관측성: 로그(ELK/Cloud Logging), 메트릭(Prometheus/Grafana), 트레이싱(OTel) 4. 보안/프라이버시:
• 데이터 분류(신앙 여부/상담 텍스트는 민감도 상)
• 암호화 at-rest/in-transit, 접근통제, 삭제/탈퇴(데이터 삭제) 플로우
